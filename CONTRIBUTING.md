### Branch Naming Convention

```
feature/module-feature-name
bugfix/module-bug-description
hotfix/critical-issue
```

Examples:
- `feature/frontend-file-upload`
- `feature/backend-data-cleaning`
- `feature/ai-domain-classifier`
- `bugfix/frontend-chart-rendering`

### Commit Message Format

```
[Module] Brief description

Detailed description (optional)

- Change 1
- Change 2
```

Examples:
- `[Frontend] Add file upload component with drag-and-drop`
- `[Backend] Implement CSV parsing with validation`
- `[AI] Add domain classification model training`

### Pull Request Process

1. Create feature branch from `develop`
2. Make your changes
3. Test thoroughly
4. Push to your branch
5. Create Pull Request to `develop`
6. Request review from team
7. Address review comments
8. Merge after approval
